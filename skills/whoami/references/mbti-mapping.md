# whoami — MBTI Mapping

## What this is

An **optional fast-path**. If the user supplies an MBTI type, it pre-fills dial
*estimates* the user then confirms. MBTI is **not a validated instrument** — it
is used only as a rough prior, never presented as science, and every value is
confirmed per-dial.

## What MBTI can and can't inform

| Dial | MBTI signal |
|---|---|
| Breadth | Yes — from S/N (primary) and J/P (secondary) |
| Rationale | Yes — from T/F |
| Warmth | Yes — from T/F |
| Challenge | Yes — from T/F |
| Initiative | None — ask Q1, Q7 |
| Depth | None — ask Q2, Q8 |

E/I has no reliable mapping to any collaboration dial — noted, unused.

## The mapping

Each letter nudges from neutral (5):

| Letter | Nudges |
|---|---|
| T | Challenge +2, Rationale +2, Warmth −2 |
| F | Challenge −2, Rationale −2, Warmth +2 |
| N | Breadth +2 |
| S | Breadth −2 |
| P | Breadth +1 |
| J | Breadth −1 |

Example — INTJ → Breadth 6, Rationale 7, Warmth 3, Challenge 7; Initiative &
Depth left at 5 (asked).

## How it's applied

The 4 informed dials get their estimate tagged **medium confidence** → shown for
confirmation in Step 7 (not cold-asked). Initiative and Depth are tagged **low**
→ their questions (Q1, Q2, Q7, Q8) are asked. Net: the fast-path confirms 4 and
asks ~4, instead of 9 cold — meaningfully faster, without overclaiming.

Confirmation is **mandatory and per-dial** — never silently accept an
MBTI-derived value. Invalid or partial type → ask for it cleanly once, or fall
back to the full interview.
