# Report Contract

The exact shape of `gamification-fit.md` (canonical) and what the HTML renders. The
Markdown is the source of truth; the HTML (`templates/gamification-fit.html`) is a
render of it.

---

## Markdown structure (headings load-bearing)

```markdown
# Gamification Fit — [Product / Feature Name]

> Generated [YYYY-MM-DD]. Goal: [the confirmed Phase 2 metric]. Ingestion lane:
> [SocratiCode | glob-grep]. Analytics: [export provided | none — repeat-usage inferred].

## Scorecard
- Analyzed: N candidates · Recommended: Y · Refused/skipped: Z
- (Z is usually the largest. That is the skill working.)

## Recommended mechanics
[Y high-conviction cards — the card shape below. If Y is 0, say so plainly: the
honest finding is "nothing here is worth gamifying yet, and here's why."]

## Deliberately NOT gamified, and why
[The Z refusals/skips — the prominent section. Each: feature → reason → (REFUSED
bar | SKIP reason). Group hard refusals (regulated/minors/compulsion) first.]

## Cross-feature checks
- Cannibalization: [...]
- Attention/notification budget assumed: [...]
- Goal contradictions: [...]

## Hand-off
[Which cards to build first; effort-ordered. Pointer to team-composer /
sub-agent-coordinator if the build needs scoping. The "feel" honesty note. The
opinion disclaimer.]
```

## Recommended card shape (per recommendation)

Every recommended card carries all of these — a missing field is a gap to fill, not
a field to drop:

1. **Feature / action** — the user-facing verb (from the inventory).
2. **Goal it serves** — the Phase 2 metric this moves. Named, not "engagement".
3. **Recommended mechanic(s)** — group (A intrinsic-supporting / B extrinsic) +
   name, from the taxonomy.
4. **SDT / behavior rationale** — which need it feeds (autonomy / competence /
   relatedness) and why this mechanic fits *this* action.
5. **Sample microcopy + tone note** — concrete example text + the tone ("a quiet
   record, never a leash"). No shame/loss framing.
6. **Effort tag** — `CSS-afternoon` (a progress ring) | `websockets-project` (live
   multiplayer) | `instrument-first` (needs tracking before it can exist).
7. **"Skip if"** — the condition under which this recommendation stops being worth
   it (e.g. "skip if fewer than ~20% of users reach this action weekly").
8. **Anti-pattern warning** — the nearest dark-pattern this mechanic could degrade
   into, and the line not to cross.
9. **Confidence** — `observed` (analytics back the repeat-usage claim) | `inferred`
   (schema/route implies it) | `unknown`. Pinned to provenance.
10. **Sunset** — *required* for any Group B (extrinsic) mechanic; omit for Group A.

## Refusal card shape (per refusal/skip)

1. **Feature / action**
2. **Verdict** — `REFUSED — <veto bar>` or `SKIP — <fit-test reason>`
3. **Reason** — one honest line. For hard refusals (regulated/minors), state the
   category and that this skill declines it.

## The opinion disclaimer (render verbatim-equivalent)

> **⚠️ Opinion, not advice.** This report is an opinionated, evidence-grounded set
> of suggestions generated by an AI tool — **not** professional, legal, or
> behavioral-health advice, and not a guarantee of any outcome. Gamification is
> dual-use; these recommendations are intended to *support* users' intrinsic
> motivation and respect their autonomy. Mechanics targeting regulated flows or
> minors are declined. The builder decides what to ship, and the *feel* of any
> mechanic depends on implementation craft this document cannot deliver.

**Banned phrasing** (refuse to ship if present in a recommendation): "will increase
[metric]", "guaranteed to boost engagement", any promise of a behavioral outcome.
Frame every recommendation as *"in our opinion"* / *"the evidence suggests"*.

## HTML render notes

- **Scorecard header** — three counts (analyzed / recommended / refused) as the
  top band, refused most prominent.
- **Variable card grid** — recommended cards (`.card.rec`) then refused cards
  (`.card.refused`), each with a confidence chip and collapsible provenance. Length
  is data-driven (3 to ~20 cards), not a fixed grid.
- **Filter** — toggle recommended / refused visibility (mirror startup-audit's tier
  filter).
- Self-contained, prints to PDF, zero network deps, no localStorage, no PII. Brand
  tokens from `DESIGN.md` if present, neutral fallback otherwise.
